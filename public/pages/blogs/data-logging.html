We programmers tend to focus all our efforts in building the application and when it's finally complete, we ship it out to the world. All of our work as paid off and we can close the books on our application until the next feature or request, right? NO. All of our applications tend to come with unexcepted behavior or issues, we defined as "bugs".

When they are notice, we programmers are quick to respond by pulling down the code to our local machines and start immediately debugging the issue. We open up the code and attempt to run the code with hopes that one of our error handlers display the issue when attempt to build, compile and/or run the application.
