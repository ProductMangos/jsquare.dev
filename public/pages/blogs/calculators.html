Building Calculators on the Web

Dec 30, 2024

As an actively working Software Developer, I have the pleasure (not always) of building a wide variety of applications and features that many people would actually use. One of those features are the calculators featured throughout the PSECU website.

A "wee bit" of history

Prior to my arrival, all the calculators were hosted through a 3rd party (some still are) that PSECU redirected to when a user clicked on a link. For some reason, either the developers weren't capable or simply not trusted to build out the calculators "in-house". So it was decided to simply redirect the user to this 3rd party website to view all the calculators that they built out for our users to use. The users didn't use the calculators.

The problem with the UI and static data

Speaking on behalf of user experience, anytime there is a visible transition from the website you intend to be on and a website you didn't know/want to be on, is a bad user experience. There are never really good ways on hiding that transition as URL will always scream otherwise.

It becomes especially bad when the 3rd party didn't even attempt to make the effort to make it appear part of the PSECU website. It was a pale screen with calculator options and the companies logo on the top left of that screen. It was as whitelabled as could be.

In addition to the poor UI, the issue of static data remained. When referring to calculators, there is always that "dynamic" element that makes that calculator more relevant to the user with connection to where that calculator is being used. An example of this... is a bank and the rate is being offered for that calculation. You'll see some calculators that provide that input into their calculators.
